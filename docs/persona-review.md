# ペルソナ別の見やすさ確認メモ

## 前提
- 対象: Claude Cowork オフ会スライド
- 観点: 文字の理解しやすさ、情報量、画像追加後の視線誘導、初心者にとってのわかりやすさ

## ペルソナ1: 生成AIほぼ初体験の会社員
- 気になる点: 英語ラベルが多いと、その場で意味を考えてしまう
- 対応: Section 3 とハンズオンのチップを日本語化
- 評価: 流れが追いやすくなった

## ペルソナ2: IT用語が苦手な事務職
- 気になる点: 「Files」「Schedule」のような単語がノイズになりやすい
- 対応: `ファイル / ブラウザ / スキル / 定期実行` に統一
- 評価: セクションの役割が一目で伝わりやすい

## ペルソナ3: 後方の席から見る参加者
- 気になる点: 画像を大きく入れすぎると、本文が圧迫されて読みにくい
- 対応: ブラウザ・スケジュール・実演スライドの画像高さと余白を圧縮
- 評価: 主メッセージを残したまま、画像も添えやすい構成にした

## ペルソナ4: 手を動かしながら参加する初心者
- 気になる点: どこを見れば次の行動につながるかが分かりづらいと止まりやすい
- 対応: ブラウザ操作のキャプションを短文化し、ハンズオン導線のチップもやさしい表現へ変更
- 評価: 「何をすればいいか」が追いやすい

## ペルソナ5: 後日スライドだけ見返す人
- 気になる点: 画像だけ大きくて説明が少ないと、文脈が思い出しづらい
- 対応: 実演スライドの画像はカード内に収め、短い補足キャプションを維持
- 評価: 見返したときも意図が残りやすい

## 今回の主な確認ポイント
- 初期ページ数表示と実際のスライド枚数が一致していること
- 画像を入れたスライドだけ専用のコンパクト調整が入っていること
- 追加したローカル画像ファイルがすべて存在していること

## 残る注意点
- 実機の表示倍率やプロジェクター解像度で、見え方はまだ少し変わる
- もし会場で文字が小さく見える場合は、まず実演スライドの補足文をさらに短くするのが安全
